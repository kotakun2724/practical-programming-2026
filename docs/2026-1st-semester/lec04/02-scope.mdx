# スコープと変数の有効範囲

## スコープとは

**スコープ（有効範囲）**は、変数がアクセスできる範囲のことです。
一度宣言した変数は、その宣言時点から最後の `}` までの範囲でのみ使用できます。

## ブロックスコープ

`{` と `}` で囲まれた領域を**ブロック**と呼び、その中で宣言した変数は
そのブロック内でのみ有効です。

```c
#include <stdio.h>

int main() {
    int x = 10;  // x は main() 内で有効
    printf("%d\n", x);  // OK: 10 と出力

    {
        int x = 20;  // 新しい x を定義（前の x と別）
        printf("%d\n", x);  // OK: 20 と出力
    }

    printf("%d\n", x);  // OK: 10 と出力（元の x）
    // int y = 30;  // ほかのブロック内で宣言した y にはアクセスできない
    
    return 0;
}
```

## 関数スコープ

関数の引数と関数内で宣言した変数は、その関数内でのみ有効です。

```c
#include <stdio.h>

void func(int a) {  // a は func() 内でのみ有効
    int b = 20;  // b は func() 内でのみ有効
    printf("%d %d\n", a, b);
}

int main() {
    int x = 10;
    func(x);  // func() を呼び出し、x を渡す

    // printf("%d\n", a);  // エラー: a は func() の外では見えない
    // printf("%d\n", b);  // エラー: b は func() の外では見えない

    return 0;
}
```

出力：`10 20`

## グローバル変数（全体スコープ）

関数の外で宣言した変数は、すべての関数からアクセスできます。

```c
#include <stdio.h>

int global_var = 100;  // グローバル変数

void func() {
    printf("%d\n", global_var);  // OK: グローバル変数にアクセス
    global_var = 200;
}

int main() {
    printf("%d\n", global_var);  // OK: 100
    func();
    printf("%d\n", global_var);  // OK: 200（func() で変更された）
    return 0;
}
```

:::warning 注意事項
グローバル変数は便利ですが、使いすぎると**プログラムの追跡が難しく**なります。
一般的には、グローバル変数は最小限に使い、
必要な値は関数の引数で渡すことが推奨されます。
:::

## スコープのベストプラクティス

1. **必要な範囲だけで宣言する** - グローバル変数は避ける
2. **関数の引数と戻り値を活用する** - データの流れを明確にする
3. **同じ名前の変数は避ける** - 異なるスコープでも混乱の原因になる

例：関数の戻り値を使った実装

```c
#include <stdio.h>

int add(int a, int b) {
    return a + b;
}

int main() {
    int x = 10, y = 20;
    int result = add(x, y);  // 関数から値を返してもらう
    printf("%d\n", result);
    return 0;
}
```

これで、`add()` 関数内での計算は独立していて、結果だけを受け取る形になります。
