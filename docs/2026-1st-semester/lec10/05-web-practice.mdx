# 演習: リクエスト操作

今回の講義で学んだパラメータ改竄とヘッダー偽装を使って、Daily AlpacaHackの問題を解いてみましょう。

## 演習 1: Alpaca Rangers 2（Easy）

**問題リンク**: [https://alpacahack.com/daily/challenges/alpaca-rangers-2](https://alpacahack.com/daily/challenges/alpaca-rangers-2)

### 問題概要

`?img=red.png` のように画像ファイルを指定して表示するWebアプリです。`../` は `replace` 関数でサニタイズされていますが、不十分です。

### 解法のヒント

1. パラメータでファイル名を指定できます
2. `../` はフィルタリングされていますが、replace関数の動作を確認しましょう
3. `....//` と入力すると、フィルタを通過した後に `../` が残ります

<details>
<summary>詳しいヒント</summary>

`replace("../", "")` は `../` を空文字列に置換します。

`....//` と入力すると：
1. `....//` → `../` （1回目の置換で `../` が削除される）
2. 結果として `../` が残る

</details>

<details>
<summary>解説</summary>

ディレクトリトラバーサルのフィルタバイパス問題です。

`replace` 関数が1回だけ実行される場合、`....//` のように入力することで、フィルタを通過した後に `../` が残ります。

```
?img=....//....//flag.txt
```

**不完全なサニタイゼーション**の危険性を示す問題です。

</details>

## 演習 2: ヘッダー偽装の練習

**ターゲット**: `https://httpbin.org/headers`

### 課題

1. 通常のリクエストを送信して、送信されたヘッダーを確認します
2. `X-Forwarded-For` ヘッダーを追加して、IPアドレスを偽装します
3. `User-Agent` を変更して、ボットになりすまします

### 解答例

```bash
curl https://httpbin.org/headers

curl https://httpbin.org/headers \
  -H "X-Forwarded-For: 192.168.1.1"

curl https://httpbin.org/headers \
  -A "Googlebot/2.1"
```
