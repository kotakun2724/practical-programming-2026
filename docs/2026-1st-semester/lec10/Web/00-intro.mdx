# はじめに

:::warning

ここで学ぶ技術は， **CTFの範囲内でのみ** 使用してください．許可なく他人のシステムを攻撃することは **不正アクセス禁止法** で禁じられています．演習問題は必ず AlpacaHack などの公式プラットフォーム上で解いてください．

:::

## 今回の目標

前回は Web CTF の **道具** と **仕組み** を学びました。今回は、その道具を使って **リクエストを改ざんする** 攻撃手法を学びます。

1. **リクエストの改ざん** — URL/フォーム/Cookie/ヘッダーなど、クライアントが送る値を書き換える攻撃を理解する
2. **代表的な脆弱性パターン** — 認可不備・IP制限回避・ディレクトリトラバーサルなど、書き換えで突ける弱点を理解する
3. 上記を踏まえて， **AlpacaHack の Daily 問題** を自力で解けるようになる

## 前回のおさらい

前回（lec09）は，以下を使えるようになることが目標でした。

### 道具: `curl` の主要オプション

| オプション | 用途 | 例 |
|:--|:--|:--|
| `-i` | レスポンスヘッダーも表示 | `curl -i https://target/` |
| `-I` | ヘッダーのみ表示 | `curl -I https://target/` |
| `-X` | メソッド変更 | `curl -X POST https://target/api` |
| `-H` | 任意のヘッダーを追加 | `curl -H "X-Forwarded-For: 127.0.0.1" …` |
| `-A` | User-Agent を変更 | `curl -A "Googlebot/2.1" …` |
| `-d` | ボディ送信（POST フォーム） | `curl -d "user=admin" …` |
| `-b` / `-c` | Cookie の送信 / 保存 | `curl -b cookies.txt …` |
| `-L` | リダイレクト追従 | `curl -L …` |
| `-sS` | エラーは表示，静かに実行 | `curl -sS …` |
| `-v` | TLS を含む詳細ログ | `curl -v …` |

### 仕組み: HTTP の登場人物

- **HTTP はステートレス** で，リクエスト単体では「誰が送ってきたか」を持たない。
- サーバーが `Set-Cookie` でブラウザに目印を渡し，ブラウザが次回以降自動で `Cookie` ヘッダーで送り返す。
- ログイン状態のような **セッション** は，Cookie（または類似の機構）で **クライアント ↔ サーバー間** で取り回される。

## 今回の立ち位置

前回は **正規のリクエストを正しく再現する** 方法を学びました。
今回は一歩進んで， **意図的にリクエストを改ざんして** サーバー側のロジック穴を突く攻撃手法を扱います。

ポイントは，

> **「クライアントから送られる値は，全部書き換えられる」**

という大前提です。これを念頭に置きながら，次のページから具体的な攻撃手法を見ていきましょう。
