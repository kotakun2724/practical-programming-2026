# HTTPヘッダー偽装

## HTTPヘッダー偽装とは

**HTTPヘッダー偽装** は、リクエストヘッダーの値を意図的に変更する攻撃手法です。サーバーのアクセス制御を回避します。

### 前提条件

この攻撃が成立するには：
- サーバーが特定のリクエストヘッダーを信頼してアクセス制御に使用している
- ヘッダーの値がクライアント側から自由に設定可能
- リバースプロキシ環境でIPアドレス取得にヘッダーを使用している

### 攻撃成功時のレスポンス例

```http
HTTP/1.1 200 OK
Content-Type: text/html

<html>
<body>
  <h1>Admin Panel</h1>
  <p>FLAG{header_manipulation_success}</p>
</body>
</html>
```

### 攻撃失敗時のエラー例

```http
HTTP/1.1 403 Forbidden
{"error": "Access from your IP is not allowed"}
```

### 成功/失敗の判定基準

- **成功**: 200 OKで保護されたコンテンツ（管理画面、flag等）が表示される
- **失敗**: 403 Forbidden、またはアクセス拒否のメッセージ

## 攻撃の対象となるヘッダー

### 1. IPアドレス関連

| ヘッダー | 説明 |
|---|---|
| `X-Forwarded-For` | クライアントの元IPアドレス |
| `X-Real-IP` | クライアントの実際のIPアドレス |

#### 使用場面

Webサーバーの前に**リバースプロキシ**（Webサーバーの前段に配置され、クライアントからのリクエストを内部サーバーに取り次ぐ中継サーバー。Nginxなど）が置かれている場合、アプリケーションがクライアントの実際のIPアドレスを知るために使用します。

#### 攻撃例

```bash
curl https://target.com/admin \
  -H "X-Forwarded-For: 127.0.0.1"
```

ローカル（`127.0.0.1`）からのアクセスのみ許可されているページを突破できることがあります。

### 2. リファラー

| ヘッダー | 説明 |
|---|---|
| `Referer` | 前のページのURL |

#### 攻撃例

```bash
curl https://target.com/flag \
  -H "Referer: https://target.com/login"
```

特定のページからの遷移のみ許可されている場合、Refererを偽装してアクセスできることがあります。

### 3. User-Agent

| ヘッダー | 説明 |
|---|---|
| `User-Agent` | ブラウザの種類・バージョン |

#### 攻撃例

```bash
curl https://target.com \
  -A "Googlebot/2.1"
```

検索エンジンのボットからのアクセスのみ許可している場合、User-Agentを偽装してアクセスできることがあります。

## 実践: ヘッダー偽装

### 複数のヘッダーを組み合わせる

```bash
curl https://target.com/admin \
  -H "X-Forwarded-For: 127.0.0.1" \
  -H "User-Agent: Googlebot/2.1"
```

## よくある脆弱性パターン

### IP制限の回避

```
もし X-Forwarded-For が "127.0.0.1" なら:
    ローカルアクセスとして許可
```

`X-Forwarded-For` ヘッダーはクライアントが自由に設定できるため、このチェックは意味がありません。

## 防御方法

### IPアドレスの取得

- `X-Forwarded-For` を信頼せず、実際の接続元IPを使用する
- リバースプロキシの設定で、`X-Forwarded-For` を上書きする

## CTFでのポイント

1. **アクセス制限がある場合**、IP関連のヘッダーを偽装してみる
2. **ボット限定のページ**がある場合、User-Agentを変更してみる
3. **特定ページからのみアクセス可能な場合**、Refererを偽装してみる

## まとめ

HTTPヘッダー偽装は、サーバーのアクセス制御ロジックの弱点を突く攻撃手法です。

特に `X-Forwarded-For` によるIP偽装は、CTFで頻出のテクニックなので覚えておきましょう。
