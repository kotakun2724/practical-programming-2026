# 計算量の考え方

## 計算量とは

**計算量**は、入力サイズ `N` に対して、プログラムがおおよそ何回の処理を行うかを表す指標です。
細かい定数や下位の項は気にせず、**`N` が大きくなったときの増え方**だけに注目するのがポイントです。

たとえば、配列の合計を求めるコード:

```c
int sum = 0;
for (int i = 0; i < n; i++) {
    sum += a[i];  // この行が n 回実行される
}
```

ループが `N` 回回るので、処理回数はおよそ `N` 回。これを `O(N)` と表します。

## ビッグオー記法

代表的な計算量を、`N` が大きくなったときの感覚と一緒に整理します。

| 計算量 | 例 | `N=10^3` | `N=10^5` | `N=10^8` |
|:-------|:---|:---------|:---------|:---------|
| `O(1)` | 配列の先頭要素を取り出す | 一瞬 | 一瞬 | 一瞬 |
| `O(log N)` | 二分探索 | 一瞬 | 一瞬 | 一瞬 |
| `O(N)` | 配列の最大値 | 一瞬 | 一瞬 | 1秒前後 |
| `O(N log N)` | マージソート | 一瞬 | 一瞬 | 数秒 |
| `O(N^2)` | 二重ループ | 一瞬 | 数秒〜十数秒 | 不可能 |

:::tip 1秒で動く計算量の目安
競技プログラミングの一般的な感覚では、1秒間に実行できるのは **約 `10^8` 回** の単純な操作です。
入力サイズ `N` と計算量から、TLE になりそうかを判断する習慣をつけましょう。
:::

## O(N) の例: 配列の最大値

配列を1回走査するだけで最大値を見つけられます。

```c
int max = a[0];
for (int i = 1; i < n; i++) {
    if (a[i] > max) max = a[i];
}
```

ループ回数は `N - 1` 回。定数倍を無視して `O(N)` です。

## O(N^2) の例: 全ペアの比較

すべての `(i, j)` ペアを調べる処理は二重ループになります。

```c
for (int i = 0; i < n; i++) {
    for (int j = i + 1; j < n; j++) {
        if (a[i] + a[j] == target) {
            // 何か処理
        }
    }
}
```

内側のループ回数の合計はおよそ `N(N-1)/2` 回。`N^2` に比例して増えるので `O(N^2)` です。

## 計算量の見積もり方

ループの**ネストの深さ**と、それぞれのループが何回回るかを掛け算するのが基本です。

| コード形 | 計算量 |
|:---------|:-------|
| 単一の `for` ループ（`N` 回）| `O(N)` |
| 二重 `for` ループ（`N` × `N`）| `O(N^2)` |
| 二重 `for` ループ（`N` × `log N`）| `O(N log N)` |
| 三重 `for` ループ（`N` × `N` × `N`）| `O(N^3)` |

:::warning AOJ で `N <= 10^5` のとき
`O(N^2)` は約 100 億回（`10^10`）の処理になり、ほぼ間違いなく **TLE** になります。
このサイズなら `O(N log N)` 以下に収まるアルゴリズムを使いましょう（マージソート、二分探索など）。
:::

## 注意: 計算量は「最悪」を見る

同じアルゴリズムでも、入力によって処理回数は変わります。
慣例的に **最悪計算量**（worst case）を計算量として表記します。

- 線形探索: 最良 `O(1)` / 平均 `O(N)` / 最悪 `O(N)`
- 挿入ソート: 最良 `O(N)`（既にソート済み）/ 最悪 `O(N^2)`（逆順）

次は、計算量を意識しながら**基本ソート**を実装してみましょう。
