# 線形探索と二分探索

配列から目的の値を**探す**処理にも、いくつかのアルゴリズムがあります。
ここでは、最も基本的な**線形探索**と、ソート済み配列に対して使える**二分探索**を学びます。

## 線形探索

### 考え方

配列の先頭から末尾まで順に見ていき、目的の値と一致したらそのインデックスを返します。
配列がソートされている必要はありません。

### 実装

```c
#include <stdio.h>

int linear_search(int *a, int n, int key) {
    for (int i = 0; i < n; i++) {
        if (a[i] == key) return i;
    }
    return -1;  // 見つからなかった
}

int main(void) {
    int a[] = {3, 1, 4, 1, 5, 9, 2, 6};
    int idx = linear_search(a, 8, 5);
    printf("%d\n", idx);  // 4
    return 0;
}
```

### C++版

```cpp
#include <iostream>
#include <vector>
using namespace std;

int linear_search(const vector<int>& a, int key) {
    for (int i = 0; i < (int)a.size(); i++) {
        if (a[i] == key) return i;
    }
    return -1;
}
```

### 計算量

| ケース | 計算量 |
|:-------|:-------|
| 最良（先頭にある） | `O(1)` |
| 平均 | `O(N)` |
| 最悪（末尾にある or 存在しない） | `O(N)` |

:::tip 個数を数えるバリエーション
AOJ `ALDS1_4_A` は「見つかった**個数**を答える」問題です。
`return i;` の代わりに `cnt++;` を続けて、最後に `cnt` を出力する形に変えるだけで対応できます。
:::

## 二分探索

### 考え方

**ソート済みの配列**であれば、毎回半分に絞り込みながら探せます。

1. 配列の真ん中の要素 `a[mid]` を見る
2. `a[mid] == key` なら見つかった
3. `a[mid] < key` なら、右半分を探す
4. `a[mid] > key` なら、左半分を探す

これを繰り返すと、毎ステップで探索範囲が半分になるので、計算量は `O(log N)` です。

### 図解

```
key = 7
[1 3 5 7 9 11 13]
       ^ mid=3 (a[3]=7)  → 見つかった！
```

```
key = 11
[1 3 5 7 9 11 13]
       ^ mid=3 (a[3]=7) < 11 → 右半分へ
[      9 11 13]
          ^ mid=5 (a[5]=11)  → 見つかった！
```

### 実装

```c
#include <stdio.h>

int binary_search(int *a, int n, int key) {
    int lo = 0, hi = n - 1;
    while (lo <= hi) {
        int mid = (lo + hi) / 2;
        if (a[mid] == key) return mid;
        else if (a[mid] < key) lo = mid + 1;
        else hi = mid - 1;
    }
    return -1;
}

int main(void) {
    int a[] = {1, 3, 5, 7, 9, 11, 13};  // ソート済み
    printf("%d\n", binary_search(a, 7, 11));  // 5
    printf("%d\n", binary_search(a, 7, 6));   // -1
    return 0;
}
```

### C++版

```cpp
#include <iostream>
#include <vector>
using namespace std;

int binary_search_idx(const vector<int>& a, int key) {
    int lo = 0, hi = (int)a.size() - 1;
    while (lo <= hi) {
        int mid = (lo + hi) / 2;
        if (a[mid] == key) return mid;
        else if (a[mid] < key) lo = mid + 1;
        else hi = mid - 1;
    }
    return -1;
}
```

### 計算量

| ケース | 計算量 |
|:-------|:-------|
| 最良 | `O(1)` |
| 平均 / 最悪 | `O(log N)` |

`N = 10^9` でも、`log_2(10^9) ≈ 30` 回程度の比較で済みます。線形探索と比べて圧倒的に高速です。

## 二分探索が成立する条件

| 条件 | 説明 |
|:-----|:-----|
| ソート済みであること | 昇順または降順に並んでいる必要がある |
| ランダムアクセスできること | 任意のインデックスを `O(1)` で読める |

:::warning ソートされていない配列に二分探索を使わない
未ソートの配列に二分探索を適用すると、**間違った結果**を返します。
あらかじめ並べ替えるか、線形探索を使いましょう。
:::

## C++ の標準ライブラリ

C++ では、二分探索を自分で書かなくても標準ライブラリで使えます。

```cpp
#include <algorithm>
#include <vector>

vector<int> a = {1, 3, 5, 7, 9};
bool found = binary_search(a.begin(), a.end(), 5);   // true
auto it = lower_bound(a.begin(), a.end(), 6);        // 7 を指すイテレータ
int idx = it - a.begin();                            // インデックスに変換
```

:::note 競技プログラミングで頻出
`lower_bound` / `upper_bound` は「指定した値以上 / より大きい要素の最初の位置」を返します。
範囲カウントや挿入位置の特定で非常に便利です。
:::
