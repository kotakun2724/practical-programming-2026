### 各問題のヒント

#### ALDS1_2_A: Bubble Sort

`if (a[j-1] > a[j])` で交換した回数をカウントします。出力は「ソート後の配列」と「交換回数」の2行です。

```c
for (int i = 0; i < n - 1; i++) {
    for (int j = n - 1; j > i; j--) {
        if (a[j - 1] > a[j]) {
            int t = a[j - 1]; a[j - 1] = a[j]; a[j] = t;
            cnt++;
        }
    }
}
```

#### ALDS1_2_B: Selection Sort

未整列部分から**最小値の位置**を見つけて、先頭と交換します。
**実際に交換が起きたとき**だけカウントするのがポイントです（`minj != i` のときのみ `cnt++`）。

```c
for (int i = 0; i < n - 1; i++) {
    int minj = i;
    for (int j = i + 1; j < n; j++) {
        if (a[j] < a[minj]) minj = j;
    }
    if (minj != i) {
        int t = a[i]; a[i] = a[minj]; a[minj] = t;
        cnt++;
    }
}
```

#### ALDS1_4_A: Linear Search

`T` の各値について、`S` を線形探索して見つかった個数を数えます。

```c
int cnt = 0;
for (int i = 0; i < q; i++) {
    int key;
    scanf("%d", &key);
    for (int j = 0; j < n; j++) {
        if (s[j] == key) { cnt++; break; }
    }
}
```

#### ALDS1_4_B: Binary Search

`S` がソート済みなので、二分探索で各 `T[i]` を `O(log N)` で判定できます。
線形探索だと TLE する可能性があるので注意しましょう。

## 困ったときは

### よくあるエラー

| エラー | 原因 | 対処法 |
|:-------|:-----|:-------|
| TLE | `O(N^2)` など計算量が大きすぎる | 二分探索やソート済み配列を活用する |
| WA | swap 回数の数え方 | 実際に交換した回数のみカウントしているか確認 |
| WA | 出力形式の誤り | 空白・改行の位置を問題文と照らし合わせる |
| CE | 文法エラー | 配列添字やループ境界を確認 |
