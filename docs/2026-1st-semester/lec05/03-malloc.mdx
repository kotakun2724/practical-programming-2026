# 動的メモリ確保

## 静的配列の問題

これまでの配列宣言では、サイズをコンパイル時に決めなければなりませんでした。

```c
int a[100];  // サイズは固定（最大 100 個）
```

入力されるデータの個数が実行時まで分からない場合、あらかじめ大きなサイズを確保しておくしかなく、メモリの無駄遣いになることがあります。

## `malloc` による動的メモリ確保

`<stdlib.h>` に含まれる `malloc` 関数を使うと、**実行時にサイズを指定**してメモリを確保できます。

```c
#include <stdlib.h>

// 構文
void *malloc(size_t size);

// 使い方
int *arr = (int *)malloc(n * sizeof(int));
```

| 引数 | 内容 |
|:-----|:-----|
| `n * sizeof(int)` | `int` 型 `n` 個分のバイト数を指定 |
| `(int *)` | 戻り値 `void *` を `int *` に型変換（キャスト） |

## 使用例

```c
#include <stdio.h>
#include <stdlib.h>

int main() {
    int n;
    scanf("%d", &n);

    int *arr = (int *)malloc(n * sizeof(int));

    for (int i = 0; i < n; i++) {
        scanf("%d", &arr[i]);
    }

    // 合計を計算
    int sum = 0;
    for (int i = 0; i < n; i++) {
        sum += arr[i];
    }
    printf("sum = %d\n", sum);

    free(arr);  // 必ず解放する
    return 0;
}
```

## `free` によるメモリ解放

`malloc` で確保したメモリは、使い終わったら必ず `free` で解放します。

```c
free(arr);
arr = NULL;  // ポインタを NULL にしておくと安全
```

:::warning メモリリークに注意
`free` を忘れると**メモリリーク**が発生し、プログラムが多くのメモリを消費し続けます。
`malloc` と `free` は常にペアで使いましょう。
:::

## `malloc` の失敗

メモリが確保できなかった場合、`malloc` は `NULL` を返します。
大きなメモリを確保するときは確認する習慣をつけましょう。

```c
int *arr = (int *)malloc(n * sizeof(int));
if (arr == NULL) {
    printf("メモリ確保に失敗しました\n");
    return 1;
}
```

## `calloc` も便利

`calloc` は確保したメモリを **0 で初期化** してくれます。

```c
// calloc(要素数, 1要素のバイト数)
int *arr = (int *)calloc(n, sizeof(int));
// arr の全要素が 0 に初期化される
```

## まとめ

| 関数 | 用途 |
|:-----|:-----|
| `malloc(size)` | 指定バイト数のメモリを確保（初期化なし） |
| `calloc(n, size)` | n 個分のメモリを確保し 0 に初期化 |
| `free(ptr)` | 確保したメモリを解放 |

次は**練習問題**に取り組みましょう。
