RSA 暗号への攻撃手法

今回は、CTF や実際のシステムでよく見られる2つの代表的な攻撃手法を解説します．

1. NNN が小さい RSA 暗号​

現代の安全な RSA 暗号では NNN に 2048 ビット（約 617 桁）以上の巨大な数が使われますが、もし NNN のビット数が小さい場合、すぐに素因数分解ができてしまいます．

素因数分解を行うアルゴリズムは主に

  • 試し割り法
  • pollard's rho 法
  • 楕円曲線法
  • 一般数体篩法

などがあげられます．

今回は，これらのアルゴリズムは扱わず，python のライブラリに頼る方向で行きます．

sympy の factorint 関数を使うことで、比較的小さな整数であればローカル環境で素因数分解を実行できます．

sympy は pycryptodome と同様に外部パッケージなので，インストールする必要があります．

SymPy のインストール

pycryptodome と同様に

pip install sympy

を実行してください．

from sympy import factorint

factors = factorint(2026)
print(factors) 
{2: 1, 1013: 1}

素因数分解の結果が辞書型で返されます．2026=21⋅101312026 = 2^{1} \cdot 1013^{1}2026=21⋅10131 であることがわかります．

素因数 p,qp, qp,q が手に入れば、通常の RSA 暗号の手順通り φ(N)=(p−1)(q−1)\varphi(N) = (p-1)(q-1)φ(N)=(p−1)(q−1) と d≡e−1 mod φ(N)d \equiv e^{-1} \bmod{\varphi(N)}d≡e−1modφ(N) を計算して，復号が行えます．

2. eee が小さすぎる RSA 暗号 Low Public Exponent Attack​

RSA 暗号の eee には、計算速度を高速化するために通常 e=65537e = 65537e=65537 （ 216+12^{16} + 1216+1 ）がよく用いられます．
二進数で表すと，10000000000000001 と最上位桁と最下位桁以外が 0 となり色々都合が良い数字です．
ちなみに 16進数で表すと 10001 です．

CTF では，この eee を極端に小さい数にして出題することがあります．

ここで、平文メッセージ mmm のサイズが小さく、公開鍵 NNN が非常に大きい場合、以下の状態が発生することがあります．

me<Nm^e < Nme<N

なにが問題であるのか？​

RSA 暗号の暗号化は

c=me mod Nc = m^{e} \bmod{N}c=memodN

であったことを思い出すと，me<Nm^{e} < Nme<N なので，c=mec = m^{e}c=me です． (me>Nm^{e} > Nme>N であれば，c=me−N⋅ac = m^{e} - N \cdot ac=me−N⋅a です．)

剰余演算  mod N\bmod{N}modN が一回も行われません． つまり、あまりの計算が発生しないため、ccc の eee 乗根 ce\sqrt[e]{c}ec​ を求めるだけで，mmm が手に入ります．

こうなると、公開鍵 NNN も秘密鍵 ddd も素因数分解も一切関係なくなります．

累乗根を求めるためのアルゴリズムはニュートン法などがありますが，素因数分解と同様に外部パッケージに頼ります．

gmpy2 の iroot 関数を用いることで，累乗根を求めることができます．

gmpy2 も sympy や pycryptodome と同様にインストールする必要があります．

gmpy2 のインストール

pip install gmpy2

を実行してください．

from gmpy2 import iroot

res = iroot(27, 3)
print(res)
$ python test.py
(mpz(3), True)

272727 の 333 乗根を計算し，底は 333 であり，272727 は完全に 333^{3}33 と一致するということがわかります．

ちなみに，272727 の 444 乗根を計算してみると，

(mpz(2), False)

が得られます．27≠2427 \ne 2^{4}27=24 なので完全一致ではなく，False と出力されているのがわかります．この 222 は近似値であり，n4n^{4}n4 の中で，272727 に一番近くなるような nnn が選択されます．
