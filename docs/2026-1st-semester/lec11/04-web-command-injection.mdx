# Command Injection

## Command Injectionとは

**Command Injection（コマンドインジェクション）** は、Webアプリケーションへの攻撃手法の一つです。入力欄に不正なOSコマンドを注入し、サーバー上で任意のコマンドを実行させます。

### 前提条件

この攻撃が成立するには：
- WebアプリケーションがOSコマンドを実行している（ping, traceroute, 画像処理等）
- 入力値がコマンドに文字列結合されている
- 適切な入力検証が行われていない
- サーバーのOSによって使用可能なコマンドが異なる（Linux: `;`, Windows: `&`）

### 攻撃成功時のレスポンス例

```http
HTTP/1.1 200 OK
Content-Type: text/plain

PING 8.8.8.8 (8.8.8.8): 56 data bytes
64 bytes from 8.8.8.8: icmp_seq=0 ttl=117 time=10.5 ms
FLAG{command_injection_success}
```

### 攻撃失敗時のエラー例

```http
HTTP/1.1 400 Bad Request
{"error": "Invalid hostname"}
```

### 成功/失敗の判定基準

- **成功**: コマンド実行結果がレスポンスに含まれる、またはflagファイルの内容が表示される
- **失敗**: 入力検証エラー、またはコマンド実行エラー

### 検出方法（防御側）

- OSコマンドの実行ログ
- 不審なプロセス起動の検出
- WAFによるコマンドパターンのブロック
- ファイルシステムの変更監視

## 攻撃の仕組み

### 脆弱なコード（擬似コード）

```
# ユーザー入力を使ってpingを実行
hostname = 入力
実行("ping -c 4 " + hostname)
```

### 攻撃例

```
入力: 8.8.8.8; cat /etc/passwd
```

### 実行されるコマンド

```bash
ping -c 4 8.8.8.8; cat /etc/passwd
```

`;` でコマンドが区切られ、`cat /etc/passwd` が実行されます。

## コマンドの区切り文字

| 文字 | 説明 | 例 |
|---|---|---|
| `;` | コマンドを順に実行 | `cmd1; cmd2` |
| `&&` | 前のコマンドが成功したら実行 | `cmd1 && cmd2` |
| `\|` | パイプ（前の出力を次の入力に） | `cmd1 \| cmd2` |
| `` ` `` | バッククォート（コマンド置換） | `` cmd1 `cmd2` `` |
| `$()` | コマンド置換 | `cmd1 $(cmd2)` |

## 実践: Command Injection

### 基本的なペイロード

```
; ls
; cat /etc/passwd
; id
; whoami
```

### バッククォートや$()を使う

```
`ls`
$(ls)
```

## よくある脆弱性パターン

### ネットワークツールの入力欄

- ping
- traceroute
- nslookup

### ファイル名にコマンドを埋め込む

```
# 擬似コード
ファイル名 = 入力
実行("cat /uploads/" + ファイル名)
```

```
入力: $(cat /etc/passwd)
```

バッククォートや`$()`を使うと、ファイル名の入力欄でもコマンドを実行できます。

## 防御方法

### OSコマンドを実行しない

最も安全な方法は、OSコマンドを実行しないことです。

### 入力値の厳密な検証

```
# 英数字とドット・ハイフンのみ許可
もし 入力 が [a-zA-Z0-9.-] にマッチしないなら:
    エラー
```

## CTFでのポイント

1. **入力欄を全て試す**: 特にファイル名、ホスト名、コマンド関連
2. **様々な区切り文字を試す**: `;`, `&&`, `|`, `` ` ``, `$()`
3. **flagの場所を探す**: `/flag.txt`, `/home/ctf/flag`, 環境変数など

## よく使うコマンド

| コマンド | 用途 |
|---|---|
| `id`, `whoami` | 現在のユーザー確認 |
| `ls` | ファイル一覧 |
| `cat` | ファイル内容表示 |
| `find` | ファイル検索 |
| `env` | 環境変数表示 |

## まとめ

Command Injectionは、サーバーの完全な制御を許す非常に危険な攻撃手法です。

OSコマンドの実行は避け、やむを得ない場合は厳密な入力検証が必要です。

次回は、XSS（Cross-Site Scripting）とCSRF（Cross-Site Request Forgery）について学びます。
