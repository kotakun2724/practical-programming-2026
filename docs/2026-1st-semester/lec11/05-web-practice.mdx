# 演習: Injection攻撃

今回の講義で学んだSQL Injectionを使って、Daily AlpacaHackの問題を解いてみましょう。

## 演習: secret-table-2（Medium）

**問題リンク**: [https://alpacahack.com/daily/challenges/secret-table-2](https://alpacahack.com/daily/challenges/secret-table-2)

### 問題概要

ログインフォームで、秘密のテーブルからflagを取得します。テーブル名やカラム名は不明ですが、SQL Injectionで情報を取得できます。

### 解法のヒント

1. ページにアクセスして、機能を観察します
2. ログインフォームにSQL Injectionを試してみます
3. `UNION SELECT` でカラム数を特定します
4. `sqlite_master` からテーブル名・カラム名を取得します

<details>
<summary>詳しいヒント</summary>

UNION Injectionを使います。

まず、`UNION SELECT null, null-- ` でカラム数を特定します（2カラム）。

次に、`sqlite_master` から全テーブルのSQL定義を取得します：

```sql
UNION SELECT group_concat(sql), null FROM sqlite_master-- 
```

これで秘密のテーブル名とカラム名が分かるので、そこからflagを取得します。

</details>

<details>
<summary>解説</summary>

SQL Injectionの基本的なテクニックを使います：

1. カラム数の特定: `UNION SELECT null, null-- `
2. テーブル情報の取得: `UNION SELECT group_concat(sql), null FROM sqlite_master-- `
3. 秘密のテーブル名・カラム名を確認
4. データの抽出: `UNION SELECT flag_column, null FROM secret_table-- `

SQLiteでは `sqlite_master` テーブルから全てのテーブル定義を取得できます。

</details>

### よくあるミス

- `UNION SELECT` のカラム数が一致しないとエラーになります。`null` の数を調整してください
- `--` の後に**半角スペース**が必要です（`-- ` ）
- SQLiteでは `sqlite_master`、MySQLでは `information_schema.tables` を使います

## まとめ

今回の演習で学んだこと：

1. **SQL Injection**でデータベースの情報を不正に取得できる
2. **`sqlite_master`**（SQLite）や **`information_schema`**（MySQL）でテーブル情報を取得できる
3. 入力値の検証が重要
