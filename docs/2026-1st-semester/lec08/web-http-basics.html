HTTPの基礎

URLの構造​

URLは、Web上のリソース（ページやAPIなど）を指し示すための文字列です。

https://example.com/path/to/page?key=value
構成要素説明
httpsプロトコル
example.comホスト名
/path/to/pageパス
key=valueクエリパラメータ（GETパラメータ）

HTTPとは​

HTTP（HyperText Transfer Protocol） は、主にWebブラウザとWebサーバー間で使われるプロトコルです。

備考

HTTPはステートレス（状態を保持しない）プロトコルです。つまり、サーバーは「さっきのログイン」と「今のリクエスト」が同じユーザーか分かりません。

そこでCookieという仕組みを使います。Cookieは、サーバーがブラウザに渡す「身分証」のようなものです。 Cookieについては第9回で解説します。

HTTPリクエスト​

クライアントがサーバーに送るメッセージをリクエストと呼びます。 皆さんがブラウザでURLを入力してページを開くと、ブラウザはサーバーにHTTPのGETリクエストを送ります。

主なHTTPメソッド​

メソッド用途例
GETデータの取得ページ表示、API呼び出し
POSTデータの送信フォーム送信、ログイン

主なリクエストヘッダー​

リクエストの付加情報を記述します。

ヘッダー説明
Hostアクセス先のホスト名
Cookieサーバーから発行されたCookie（身分証）
Content-Typeリクエストボディの形式

リクエストの例​

GET /index.html HTTP/1.1
Host: example.com
Cookie: session_id=abc123

HTTPレスポンス​

サーバーがクライアントに返すメッセージをレスポンスと呼びます。

主なステータスコード​

コード意味説明
200OK成功
302Foundリダイレクト（別のページに誘導）
403Forbiddenアクセス禁止
404Not Foundページが見つからない
500Internal Server Errorサーバーのエラー

主なレスポンスヘッダー​

ヘッダー説明
Content-Typeレスポンスの形式（HTML、JSONなど）
Set-CookieCookieの発行（身分証の配布）
Locationリダイレクト先

レスポンスの例​

HTTP/1.1 200 OK
Content-Type: text/html
Set-Cookie: session_id=xyz789

<html>...</html>

CTFでのポイント​

Web問題では、以下の点を意識しましょう

  1. リクエストを観察する: ブラウザがどんなリクエストを送っているか確認する
  2. レスポンスを確認する: ステータスコード、ヘッダー、ボディを詳しく見る
  3. パラメータを操作する: URLのパラメータを書き換えてみる
  4. Cookieを確認する: どのようなCookieが設定されているか確認する
