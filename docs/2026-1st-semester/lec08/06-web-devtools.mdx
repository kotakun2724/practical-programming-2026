# Web開発者ツールとcurlコマンドの基礎

## 開発者ツールとは

ブラウザには、Webページの詳細な情報を確認できる**開発者ツール**が組み込まれています。

Web問題では、このツールを使ってHTTPリクエスト/レスポンスの確認や、ページの解析を行います。

## 起動方法

| ブラウザ | ショートカット |
|---|---|
| Chrome / Edge | `F12` または `Ctrl+Shift+I` (Windows) / `Cmd+Opt+I` (Mac) |
| Firefox | `F12` または `Ctrl+Shift+I` (Windows) / `Cmd+Opt+I` (Mac) |
| Safari | `Cmd+Opt+I` (環境設定で「開発」メニューを有効にする必要あり) |

## Elements（要素）パネル

ページのHTML構造を確認できます。

### 使い方

1. `F12` で開発者ツールを開く
2. **Elements**タブをクリック
3. HTMLのタグ構造が表示される

## Network（ネットワーク）パネル

HTTPリクエスト/レスポンスの詳細を確認できます。

### 使い方

1. `F12` で開発者ツールを開く
2. **Network**タブをクリック
3. ページをリロード（`F5`）
4. リクエストの一覧が表示される
5. 対象のリクエスト（通常は一番上）をクリック
6. 右側のパネルで詳細を確認:
   - **Headersタブ**: リクエストヘッダー、レスポンスヘッダー、ステータスコード
   - **Responseタブ**: レスポンスのボディ（HTMLなど）
   - **Cookiesタブ**: そのリクエストで送受信されたCookie

## Application（アプリケーション）パネル

Cookieなどのストレージ情報を確認できます。

### 使い方

1. `F12` で開発者ツールを開く
2. **Application**タブをクリック
3. 左側のメニューから **Cookies** を選択
4. サイトのCookie一覧を確認

## Console（コンソール）パネル

JavaScriptを実行したり、ログを確認したりできます。

```javascript
document.cookie  // Cookieの確認
```

## まとめ

開発者ツールは、Web CTFを解く上で**必須のツール**です。

特に**Networkパネル**と**Elementsパネル**は頻繁に使うので、使い方に慣れておきましょう。

## curlコマンドの基礎

開発者ツールに加えて、**curl**コマンドでもHTTPリクエストを確認できます。

```bash
# 基本的なGETリクエスト
curl https://example.com

# レスポンスヘッダーとボディを両方表示
curl -i https://example.com

# 詳細な情報（リクエスト/レスポンスの生データ）を表示
curl -v https://example.com
```

`-v` オプションを使うと、HTTPリクエストとレスポンスの生データを確認できます。CTFでは非常に便利なコマンドなので、使い方に慣れておきましょう。

次回は、このcurlコマンドについて詳しく学びます。
