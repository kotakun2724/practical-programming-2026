# HTTPの基礎

## HTTPとは

**HTTP（HyperText Transfer Protocol）**は、WebブラウザとWebサーバーの間でデータをやり取りするためのプロトコル（約束事）です。

C言語で言うところの「通信プロトコル」と同じで、「こういう形式でデータを送ります」という決まり事です。

## Cookieとは（先に知っておくべき概念）

HTTPは**ステートレス**（状態を保持しない）プロトコルです。つまり、サーバーは「さっきのログイン」と「今のリクエスト」が同じユーザーか分かりません。

そこで**Cookie**という仕組みを使います。Cookieは、サーバーがブラウザに渡す「身分証」のようなものです。

```
1. ブラウザ → サーバー: 「ログインしたい」
2. サーバー → ブラウザ: 「はい、この身分証（Cookie）を渡します」
   Set-Cookie: session_id=abc123
3. ブラウザ → サーバー: 「次のリクエストです」（身分証を自動的に提示）
   Cookie: session_id=abc123
4. サーバー: 「abc123のユーザーだね、ログイン済みと認識します」
```

このCookieの仕組みは、Web CTFの多くの問題で重要になります。

## HTTPリクエスト

ブラウザがサーバーに送るメッセージを**リクエスト**と呼びます。

### リクエストの例

```http
GET /index.html HTTP/1.1
Host: example.com
Cookie: session_id=abc123
```

### 主なHTTPメソッド

| メソッド | 用途 | 例 |
|---|---|---|
| `GET` | データの取得 | ページ表示、API呼び出し |
| `POST` | データの送信 | フォーム送信、ログイン |

### リクエストヘッダー

リクエストの付加情報を記述します。

| ヘッダー | 説明 |
|---|---|
| `Host` | アクセス先のホスト名 |
| `Cookie` | サーバーから発行されたCookie（身分証） |
| `Content-Type` | リクエストボディの形式 |

## HTTPレスポンス

サーバーがブラウザに返すメッセージを**レスポンス**と呼びます。

### レスポンスの例

```http
HTTP/1.1 200 OK
Content-Type: text/html
Set-Cookie: session_id=xyz789

<html>...</html>
```

### 主なステータスコード

| コード | 意味 | 説明 |
|---|---|---|
| `200` | OK | 成功 |
| `302` | Found | リダイレクト（別のページに誘導） |
| `403` | Forbidden | アクセス禁止 |
| `404` | Not Found | ページが見つからない |
| `500` | Internal Server Error | サーバーのエラー |

### レスポンスヘッダー

| ヘッダー | 説明 |
|---|---|
| `Content-Type` | レスポンスの形式（HTML、JSONなど） |
| `Set-Cookie` | Cookieの発行（身分証の配布） |
| `Location` | リダイレクト先 |

## URLの構造

```
https://example.com/path/to/page?key=value
```

| 構成要素 | 説明 |
|---|---|
| `https` | プロトコル |
| `example.com` | ホスト名 |
| `/path/to/page` | パス |
| `key=value` | クエリパラメータ（GETパラメータ） |

## CTFでのポイント

Webチャレンジでは、以下の点を意識しましょう：

1. **リクエストを観察する**: ブラウザがどんなリクエストを送っているか確認する
2. **レスポンスを確認する**: ステータスコード、ヘッダー、ボディを詳しく見る
3. **パラメータを操作する**: URLのパラメータを書き換えてみる
4. **Cookieを確認する**: どのようなCookieが設定されているか確認する
