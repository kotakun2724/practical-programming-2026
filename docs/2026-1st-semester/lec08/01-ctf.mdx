# CTF

CTF とは **C**apture **T**he **F**lag の頭文字を取ったもので，セキュリティ系のコンテストです．

情報セキュリティに関する問題を解くことで，知識・技術を競い合う競技です．

常設型 CTF とイベント型 CTF が存在し，本講義で使用する AlpacaHack や CryptoHack は常設型 CTF に当たります．イベント型 CTF の例としては世界で一番規模が大きいとされる DEFCON や国内で一番規模が大きい SECCON などがあります．

- 常設型 CTF $\cdots$ オンライン上に問題が公開されており，問題をいつでも解くことができます．
- イベント型 CTF $\cdots$ 期間（6h ~ 24h など）が決まっており，期間内により高いスコアを獲得することを目標とします．

## 競技形式

CTF にはいくつかの形式が存在します．

### Jeopardy 形式

オーガナイザから与えられた複数の問題を自由に解いていき，最終的なスコアを競う形式です．\
flag と呼ばれる文字列を取得することを目標とします．

今回用いる AlpacaHack などは jeopardy 形式で，ほとんどの CTF がこの形式を採用しています．

### Attack & Defense 形式

各チーム間で攻防を行うことで得点できる形式です．

### King of the Hill 形式

問題を攻略した状態を維持することで得点できる形式です．

以降は，jeopardy 形式に絞ります．

## Crypto

Cryptography は暗号の解読技術を問います．

- 古典暗号
- AES
- ChaCha20
- RSA 暗号
- 楕円曲線暗号
- 耐量子計算機暗号

などに関連する問題が多く存在します．

## Web

Web は Web アプリケーションの脆弱性を突いて flag を取得します．

- ディレクトリトラバーサル
- SQL インジェクション
- XSS

などの攻撃方法が存在します．

## others (Pwnable, Reversing, OSINT, ...)

- Pwnable $\cdots$ バイナリの脆弱性を突いてプログラムを制御する．
- Reversing $\cdots$ 逆アセンブル・逆コンパイルでプログラムの挙動を解析する．
- OSINT $\cdots$ 合法的に入手できる，公開されている情報を収集・分析し，価値ある知見を導き出す．
- Forensic $\cdots$ 通信のログやディスクイメージ，メモリダンプなどの解析を行う．

など，多くのカテゴリーが存在します．
