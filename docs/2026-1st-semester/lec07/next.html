各問題のヒント​

ALDS1_3_A: Stack​

スペース区切りのトークンを順に読み、数値なら push、演算子なら2回 pop して計算結果を push します。

if (s[0] == '+' || s[0] == '-' || s[0] == '*') {
    int b = pop();
    int a = pop();
    if (s[0] == '+') push(a + b);
    else if (s[0] == '-') push(a - b);
    else push(a * b);
} else {
    push(atoi(s));
}

最後に pop() した値が答えです。b と a の順序を間違えると - や / で WA になるので注意。

ALDS1_5_A: Exhaustive Search​

整数集合 A から いくつかの要素を選んで ターゲット m を作れるかを、再帰で全探索します。 「i 番目を使う / 使わない」の2択を再帰でたどるのが定番です。

int solve(int i, int m) {
    if (m == 0) return 1;
    if (i >= n) return 0;
    if (solve(i + 1, m)) return 1;          // i を使わない
    if (solve(i + 1, m - a[i])) return 1;   // i を使う
    return 0;
}

ALDS1_5_B: Merge Sort​

番兵 INT_MAX を使ったマージで実装するのが楽です。 比較回数は マージ中の for ループ1回ごとに1回 カウントします。

for (int k = left; k < right; k++) {
    cnt++;
    if (L[i] <= R[j]) a[k] = L[i++];
    else              a[k] = R[j++];
}

L, R のサイズは N/2 + 1 以上を必ず確保しましょう。

よくあるエラー​

エラー原因対処法
RE（Runtime Error）再帰が深すぎてスタックオーバーフローベースケースが効いているか、引数が小さくなっているかを確認
RE配列サイズ不足（マージソートの L, R 等）N/2 + 1 以上を確保する
WA演算子の順序ミス（RPN）b = pop(); a = pop(); の順を確認
WA比較回数の数え方for ループ1回ごとにカウントしているか確認
TLE再帰呼び出しが指数的に増えている同じ部分問題を何度も解いていないか確認（後の回で扱う「メモ化」で解消）
