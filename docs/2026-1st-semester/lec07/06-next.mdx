### 各問題のヒント

#### ALDS1_3_A: Stack

スペース区切りのトークンを順に読み、数値なら `push`、演算子なら2回 `pop` して計算結果を `push` します。

```c
if (s[0] == '+' || s[0] == '-' || s[0] == '*') {
    int b = pop();
    int a = pop();
    if (s[0] == '+') push(a + b);
    else if (s[0] == '-') push(a - b);
    else push(a * b);
} else {
    push(atoi(s));
}
```

最後に `pop()` した値が答えです。`b` と `a` の順序を間違えると `-` や `/` で WA になるので注意。

#### ALDS1_5_A: Exhaustive Search

整数集合 `A` から **いくつかの要素を選んで** ターゲット `m` を作れるかを、再帰で全探索します。
「`i` 番目を使う / 使わない」の2択を再帰でたどるのが定番です。

```c
int solve(int i, int m) {
    if (m == 0) return 1;
    if (i >= n) return 0;
    if (solve(i + 1, m)) return 1;          // i を使わない
    if (solve(i + 1, m - a[i])) return 1;   // i を使う
    return 0;
}
```

#### ALDS1_5_B: Merge Sort

番兵 `INT_MAX` を使ったマージで実装するのが楽です。
比較回数は **マージ中の for ループ1回ごとに1回** カウントします。

```c
for (int k = left; k < right; k++) {
    cnt++;
    if (L[i] <= R[j]) a[k] = L[i++];
    else              a[k] = R[j++];
}
```

`L`, `R` のサイズは `N/2 + 1` 以上を必ず確保しましょう。

## 困ったときは

### よくあるエラー

| エラー | 原因 | 対処法 |
|:-------|:-----|:-------|
| RE（Runtime Error） | 再帰が深すぎてスタックオーバーフロー | ベースケースが効いているか、引数が小さくなっているかを確認 |
| RE | 配列サイズ不足（マージソートの `L`, `R` 等） | `N/2 + 1` 以上を確保する |
| WA | 演算子の順序ミス（RPN） | `b = pop(); a = pop();` の順を確認 |
| WA | 比較回数の数え方 | `for` ループ1回ごとにカウントしているか確認 |
| TLE | 再帰呼び出しが指数的に増えている | 同じ部分問題を何度も解いていないか確認（後の回で扱う「メモ化」で解消） |
