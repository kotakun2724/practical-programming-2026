分割統治法とマージソート・クイックソート

分割統治法とは​

分割統治法（divide and conquer） は、問題を解く設計手法のひとつで、次の3ステップから成ります。

  1. 分割（Divide）: 問題を小さな部分問題に分ける
  2. 解決（Conquer）: 部分問題を再帰的に解く
  3. 統合（Combine）: 部分問題の答えを組み合わせて、元の問題の答えを得る
        問題(N)
       /        \
   問題(N/2)   問題(N/2)
    /    \      /    \
   ...   ...  ...   ...      <- 再帰的に分割
   |     |    |     |
   解   解    解    解         <- ベースケース（1要素）
    \   /      \   /
    統合         統合           <- 部分解を組み合わせる
       \         /
         全体の解

このパターンの典型例が、マージソート と クイックソート です。 どちらも平均 O(N log N) で、基本ソート（O(N^2)）に比べて大幅に高速です。

マージソート​

考え方​

  1. 配列を半分に分割する
  2. それぞれを再帰的にソートする
  3. 2つのソート済み配列を**マージ（併合）**して1つに戻す
[5 2 4 6 1 3]
 ├── [5 2 4]            分割
 │    ├── [5]
 │    └── [2 4]
 │         ├── [2]
 │         └── [4]
 │         ↑↓ マージ → [2 4]
 │    ↑↓ マージ → [2 4 5]
 └── [6 1 3]
      ├── [6]
      └── [1 3]
           ├── [1]
           └── [3]
           ↑↓ マージ → [1 3]
      ↑↓ マージ → [1 3 6]
 ↑↓ マージ → [1 2 3 4 5 6]

実装（C）​

#include <stdio.h>
#include <limits.h>

void merge(int *a, int left, int mid, int right) {
    int n1 = mid - left;
    int n2 = right - mid;
    int L[100001], R[100001];

    for (int i = 0; i < n1; i++) L[i] = a[left + i];
    for (int i = 0; i < n2; i++) R[i] = a[mid + i];
    L[n1] = INT_MAX;   // 番兵
    R[n2] = INT_MAX;

    int i = 0, j = 0;
    for (int k = left; k < right; k++) {
        if (L[i] <= R[j]) a[k] = L[i++];
        else              a[k] = R[j++];
    }
}

void merge_sort(int *a, int left, int right) {
    if (right - left <= 1) return;   // ベースケース（要素1個以下）
    int mid = (left + right) / 2;
    merge_sort(a, left, mid);
    merge_sort(a, mid, right);
    merge(a, left, mid, right);
}

int main(void) {
    int a[] = {5, 2, 4, 6, 1, 3};
    int n = 6;
    merge_sort(a, 0, n);
    for (int i = 0; i < n; i++) printf("%d ", a[i]);
    printf("\n");
    return 0;
}

計算量​

観点値
分割の深さlog_2 N
各深さでのマージ全体のコストO(N)
合計O(N log N)

:::tip 番兵（sentinel）テクニック 配列末尾に INT_MAX を入れておくと、マージのループで「片方が尽きたかどうか」の判定が不要になります。 AOJ ALDS1_5_B（Merge Sort）でもこの書き方が想定解として紹介されています。 :::

C++版​

#include <iostream>
#include <vector>
#include <climits>
using namespace std;

long long compare_count = 0;

void merge(vector<int>& a, int left, int mid, int right) {
    vector<int> L(a.begin() + left, a.begin() + mid);
    vector<int> R(a.begin() + mid, a.begin() + right);
    L.push_back(INT_MAX);
    R.push_back(INT_MAX);

    int i = 0, j = 0;
    for (int k = left; k < right; k++) {
        compare_count++;
        if (L[i] <= R[j]) a[k] = L[i++];
        else              a[k] = R[j++];
    }
}

void merge_sort(vector<int>& a, int left, int right) {
    if (right - left <= 1) return;
    int mid = (left + right) / 2;
    merge_sort(a, left, mid);
    merge_sort(a, mid, right);
    merge(a, left, mid, right);
}

マージソートの特徴​

項目値
最良 / 平均 / 最悪すべて O(N log N)
メモリ追加で O(N) の作業領域が必要
安定性あり（同値なら左側を優先することで実現）

クイックソート​

考え方​

  1. 配列から ピボット（基準値）を1つ選ぶ
  2. ピボットより小さい要素を左、大きい要素を右に集める（パーティション）
  3. 左右それぞれを再帰的にクイックソート

マージソートは「分割は単純、統合（マージ）が本体」でしたが、クイックソートは「分割（パーティション）が本体、統合は不要」という対照的な構造を持ちます。

パーティション（AOJ ALDS1_6_B 想定）​

配列末尾の値をピボットとして、ピボットより小さい要素を左に寄せます。

int partition(int *a, int p, int r) {
    int x = a[r];           // ピボット
    int i = p - 1;
    for (int j = p; j < r; j++) {
        if (a[j] <= x) {
            i++;
            int t = a[i]; a[i] = a[j]; a[j] = t;
        }
    }
    int t = a[i + 1]; a[i + 1] = a[r]; a[r] = t;
    return i + 1;           // ピボットの最終位置
}

クイックソート本体​

void quick_sort(int *a, int p, int r) {
    if (p < r) {
        int q = partition(a, p, r);
        quick_sort(a, p, q - 1);
        quick_sort(a, q + 1, r);
    }
}

呼び出し方は quick_sort(a, 0, n - 1); です（r は配列の末尾インデックス）。

計算量​

ケース計算量
最良O(N log N)（ピボットがほぼ中央値）
平均O(N log N)
最悪O(N^2)（ピボットが常に最小 or 最大）

:::warning 最悪 O(N^2) になる入力 すでにソート済みの配列に対し、末尾をピボットとして使うと最悪計算量に陥ります。 実用上はランダムに選んだ要素や、先頭・中央・末尾の中央値をピボットにすると安定します。 :::

マージソート vs クイックソート​

項目マージソートクイックソート
平均O(N log N)O(N log N)
最悪O(N log N)O(N^2)
追加メモリO(N)O(log N)（再帰のみ）
安定性ありなし（実装次第）
キャッシュ効率比較的低い高い
使われる場面安定性が必要なとき / 最悪を抑えたいとき平均的に最速を狙いたいとき

C++ の std::sort は内部でクイックソートを基本としつつ、再帰が深くなった場合にヒープソートへ切り替える イントロソート という改良版を採用しています。

分割統治の他の応用​

アルゴリズム内容
二分探索ソート済み配列を半分に分けて探す（O(log N)）
最近点対平面上の N 点で最も近い2点（O(N log N)）
カラツバ法大きな整数の乗算（O(N^1.58)）
高速フーリエ変換（FFT）多項式乗算（O(N log N)）
