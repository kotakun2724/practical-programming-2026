# 再帰関数の書き方

## 再帰とは

**再帰（recursion）** とは、関数が**自分自身を呼び出す**プログラミングテクニックです。
「より小さな同じ問題」を解いた結果を使って、元の問題を解きます。

## 再帰関数の2つの要素

すべての再帰関数は、次の2つから構成されます。

1. **ベースケース（base case）**: それ以上分割せずに答えを返す条件
2. **再帰ケース（recursive case）**: より小さな引数で自分自身を呼び出す

```c
return_type f(arg) {
    if (ベースケース) {
        return 直接の答え;       // 再帰の停止
    }
    return f(より小さいarg) で計算した結果を使った式;
}
```

:::warning ベースケースを忘れると無限再帰
ベースケースがないと関数は永遠に自分自身を呼び続け、**スタックオーバーフロー**でクラッシュします。
:::

## 例 1: 階乗（factorial）

数学の定義に忠実な再帰の典型例です。

```
n! = 1                  (n <= 1)
n! = n * (n - 1)!       (n >= 2)
```

```c
#include <stdio.h>

int factorial(int n) {
    if (n <= 1) return 1;             // ベースケース
    return n * factorial(n - 1);       // 再帰ケース
}

int main(void) {
    printf("%d\n", factorial(5));  // 120
    return 0;
}
```

### 実行の流れ

```
factorial(5)
 → 5 * factorial(4)
       → 4 * factorial(3)
             → 3 * factorial(2)
                   → 2 * factorial(1)
                         → 1            ベースケースで停止
                   → 2 * 1 = 2
             → 3 * 2 = 6
       → 4 * 6 = 24
 → 5 * 24 = 120
```

## 例 2: フィボナッチ数列

フィボナッチ数列 `1, 1, 2, 3, 5, 8, 13, ...` は2項を足し合わせて作ります。

```
fib(n) = n                          (n <= 1)
fib(n) = fib(n - 1) + fib(n - 2)    (n >= 2)
```

```c
int fib(int n) {
    if (n <= 1) return n;
    return fib(n - 1) + fib(n - 2);
}
```

:::warning 単純な再帰は遅い
このコードは `fib(40)` 程度で目に見えて遅くなります。
同じ引数に対して何度も同じ計算を繰り返しているためです（計算量 `O(2^N)`）。
後の回で学ぶ **メモ化** や **動的計画法** で `O(N)` に高速化できます。
:::

## 例 3: ユークリッドの互除法（再帰版）

前回はループで書きましたが、再帰でも自然に書けます。

```c
int gcd(int a, int b) {
    if (b == 0) return a;          // ベースケース
    return gcd(b, a % b);           // 再帰ケース
}
```

ループ版より**「定義そのもの」**に近く、読みやすいのが特徴です。

## 例 4: 配列の総和

配列のような「列」の処理も再帰で書けます。

```c
int sum(int *a, int n) {
    if (n == 0) return 0;
    return a[n - 1] + sum(a, n - 1);
}
```

## 再帰の落とし穴

### 1. ベースケースを忘れる

```c
int bad(int n) {
    return n + bad(n - 1);  // 停止しない！
}
```

### 2. 引数が小さくならない

```c
int bad(int n) {
    if (n == 0) return 0;
    return bad(n);  // n が減らない → 無限再帰
}
```

### 3. スタックオーバーフロー

`n` が大きすぎる（たとえば `10^6` など）と、コールスタックが溢れてプログラムが落ちます。
深さが大きくなる場合は、ループで書き直すか、末尾再帰最適化を意識する必要があります。

## 再帰で考えるコツ

| ステップ | やること |
|:---|:---|
| 1. ベースケース | 「最も小さい入力」のときの答えを決める |
| 2. 仮定 | `f(N-1)` などが既に正しく動くと**仮定**する |
| 3. 統合 | 仮定した結果を使って `f(N)` の答えを作る |

数学の**帰納法**と同じ考え方です。
「自分の関数は正しく動く」と信じて、1ステップ小さい問題に任せましょう。

## C++版のサンプル

```cpp
#include <iostream>
using namespace std;

long long factorial(int n) {
    if (n <= 1) return 1;
    return (long long)n * factorial(n - 1);
}

int gcd(int a, int b) {
    return b == 0 ? a : gcd(b, a % b);
}

int main() {
    cout << factorial(10) << '\n';  // 3628800
    cout << gcd(24, 36) << '\n';    // 12
    return 0;
}
```

次は、再帰を最大限に活かす設計手法 **分割統治法** とその代表例である **マージソート / クイックソート** を見ていきます。
