# 演習: XSSとCSRF

今回の講義で学んだXSSを使って、Daily AlpacaHackの問題を解いてみましょう。

## 演習: Alert my Flag（Medium）

**問題リンク**: [https://alpacahack.com/daily/challenges/alert-my-flag](https://alpacahack.com/daily/challenges/alert-my-flag)

### 問題概要

`username` クエリパラメータがHTMLにそのまま埋め込まれるXSS脆弱性があります。ただし、`flag` と `alert` の文字列がフィルターでブロックされています。

### 解法のヒント

1. ページにアクセスして、`username` パラメータにXSSペイロードを入力します
2. `alert` と `flag` がブロックされています
3. `<script>` タグ内で文字列連結を使ってフィルターをバイパスできます

<details>
<summary>詳しいヒント</summary>

`username` はHTML内に直接埋め込まれるため、`<script>` タグを注入します：

```
username=<script>eval('ale'+'rt(fl'+'ag')</script>
```

または、Base64エンコードを使います：

```
username=<script>eval(atob('YWxlcnQoZmxhZyk='))</script>
```

`YWxlcnQoZmxhZyk=` は `alert(flag)` のBase64エンコードです。

</details>

<details>
<summary>解説</summary>

Reflected XSSのフィルターバイパス問題です。

`username` はHTMLコンテキストに埋め込まれるため、`<script>` タグを注入してJavaScriptを実行させます。

`alert` や `flag` が直接ブロックされていても、文字列連結やBase64エンコードを使うことでフィルターを回避できます。

- `<script>eval('ale'+'rt(fl'+'ag')</script>` : 文字列連結
- `<script>eval(atob('YWxlcnQoZmxhZyk='))</script>` : Base64デコード + eval

</details>
