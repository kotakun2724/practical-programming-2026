# 演習: XSSとCSRF

今回の講義で学んだXSSとCSRFを使って、Daily AlpacaHackの問題を解いてみましょう。

## 演習 1: Alert my Flag（Medium）

**問題リンク**: [https://alpacahack.com/daily/challenges/alert-my-flag](https://alpacahack.com/daily/challenges/alert-my-flag)

### 問題概要

`username` クエリパラメータがHTMLにそのまま埋め込まれるXSS脆弱性があります。ただし、`flag` と `alert` の文字列がフィルターでブロックされています。

### 解法のヒント

1. ページにアクセスして、`username` パラメータにXSSペイロードを入力します
2. `alert` と `flag` がブロックされています
3. `<script>` タグ内で文字列連結を使ってフィルターをバイパスできます

<details>
<summary>詳しいヒントと解説</summary>

Reflected XSSのフィルターバイパス問題です。

`username` はHTMLコンテキストに埋め込まれるため、`<script>` タグを注入してJavaScriptを実行させます。

`alert` や `flag` が直接ブロックされていても、文字列連結やBase64エンコードを使うことでフィルターを回避できます：

- `<script>eval('ale'+'rt(fl'+'ag')</script>` : 文字列連結
- `<script>eval(atob('YWxlcnQoZmxhZyk='))</script>` : Base64デコード + eval

`YWxlcnQoZmxhZyk=` は `alert(flag)` のBase64エンコードです。

**よくあるミス**:

- `alert` や `flag` をそのまま入力するとブロックされます
- URLエンコードが必要な場合があります（`<` → `%3C`、`>` → `%3E`）

</details>

## 演習 2: Nano Services（Medium）

**問題リンク**: [https://alpacahack.com/daily/challenges/nano-services](https://alpacahack.com/daily/challenges/nano-services)

### 問題概要

XSS + Admin Bot の問題です。WebアプリケーションにXSS脆弱性があり、Admin Botが指定されたURLにアクセスします。Admin BotのCookieにflagが含まれていますが、Cookieには `HttpOnly` フラグが設定されているため、`document.cookie` では窃取できません。

### 解法のヒント

1. Admin Botが指定URLにアクセスし、Cookie（flag）を持っている
2. Cookieには `HttpOnly` が設定されているため、通常のXSSではCookieを窃取できない
3. `func` パラメータで3つの機能を選べる — それだけで本当に全部？
4. HttpOnlyを回避する別の方法を考える

<details>
<summary>詳しいヒントと解説</summary>

XSS + Admin Bot + HttpOnly Cookieバイパスの複合問題です。

ポイント：

1. **HttpOnly Cookieの回避**: `document.cookie` は使えませんが、XSSで `fetch()` を実行すれば、ブラウザが自動的にCookieを付与してリクエストを送信します
2. **`func` パラメータの探索**: 表示されている3つの選択肢以外にも、隠された機能がある可能性があります。ソースコードを確認して、全ての `func` の値を調べましょう
3. **間接的なflag取得**: サーバー側のAPIを呼び出し、そのレスポンスを外部サーバーに送信することでflagを取得します

この問題は、**XSSが単なる `alert()` の実行ではなく、実際の攻撃ではどのように使われるか** を示す好例です。Admin Botの権限を悪用して、XSS経由でサーバー側の機能にアクセスする、という攻撃チェーンを理解することが重要です。

</details>

## まとめ

1. **XSS** で他のユーザーのブラウザで任意のスクリプトを実行できる
2. **反射型XSS** と **保存型XSS** の2種類がある
3. **フィルターバイパス**（文字列連結、Base64）で制限を回避できる
4. **CSRF** で認証済みユーザーに意図しない操作を実行させられる
5. **HttpOnly Cookie** はXSSでのCookie窃取を防ぐが、`fetch()` 等による間接的な攻撃は防げない
6. 防御の基本は **出力のエスケープ / CSP / CSRFトークン / SameSite Cookie**
7. **クライアント側のデータもサーバー側のデータも、どちらも信頼できない**（lec10からの継続テーマ）
