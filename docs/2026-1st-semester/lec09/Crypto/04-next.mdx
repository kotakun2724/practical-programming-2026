#### 2. Extended GCD

$ax + by = \gcd(a, b)$ を満たす整数 $x, y$ を求めます．問題では $x$ と $y$ のうち、**値が小さい方**を答える必要があります．

**実装のヒント（再帰によるアプローチ）:**

拡張ユークリッドの互除法は、再帰関数を使うと綺麗に書けます．
ベースケースとして、`a == 0` の場合は $\gcd(0, b) = b$ なので、 $0 \cdot x + b \cdot 1 = b$ より $x=0, y=1$ となります．

$a$ を $b$ で割った商を $q$，余りを $r$ とすると，$r = a - bq$ となります．

ここで，`extended_gcd(b, r)` の結果がわかっているとします．そうすると，

$$
\begin{align}
    \gcd(b, r) &= bx' + ry' \\
               &= bx' + (a - bq)y' \\
               &= ay' + b(x' - qy') = \gcd(a, b)
\end{align}
$$

`extended_gcd(b, r)` の結果がわかっているので $x', y'$ はもうわかっています．

## 困ったときは

### よくあるエラー

| エラー | 原因 | 対処法 |
|:-------|:-----|:-------|
| `RecursionError` | 再帰が深すぎる | 非常に大きな数を扱う場合、ループによる実装を検討するか `sys.setrecursionlimit` を調整する（今回の課題規模なら再帰で問題ありません） |
| `TypeError` | 演算の型の不一致 | Python 3 の `/` は小数になります。整数同士の割り算（商）を求めたい場合は `//` を使いましょう。 |
| Flag が通らない | 出力形式ミス | $x$ と $y$ のうち「小さい方」を答えているか再確認してください。負の数の方が小さいことに注意。 |
