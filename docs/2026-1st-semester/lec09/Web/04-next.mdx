# 演習: ツールとCookie

今回の講義で学んだcurl、Cookie／Sessionの知識を使って、Daily AlpacaHackの問題を解いてみましょう。

## 演習 1: Message For You（Easy）

**問題リンク**: [https://alpacahack.com/daily/challenges/message-for-you](https://alpacahack.com/daily/challenges/message-for-you)

### 問題概要

Flaskアプリで、`GET /` で発行されるsessionクッキーにflagが含まれています。

### 解法のヒント

1. ページにアクセスします
2. ブラウザの開発者ツールでCookieを確認します
3. Flaskのsession cookieは `Payload.Timestamp.Signature` の形式です
4. Payload部分をBase64デコードし、さらにzlibで展開してみましょう

<details>
<summary>詳しいヒントと解説</summary>

Flaskのsession cookieは、以下の形式で構成されています：

```
Payload.Timestamp.Signature
```

Payload部分はURL-safe Base64エンコードされており、さらにzlib圧縮されています。`Signature` 部分はサーバ側の秘密鍵で計算された HMAC で、**改竄するとサーバに弾かれます**。ただし**ペイロード部分はクライアント側でも読める**ため、flag が平文で含まれていれば読み出せます。

Pythonでデコードする場合：

```python
import base64
import zlib

cookie = "eyJ...（Payload部分）"
# URL-safe Base64のパディング復元
padding = 4 - len(cookie) % 4
if padding != 4:
    cookie += "=" * padding
# デコード + 展開
payload = base64.urlsafe_b64decode(cookie)
print(zlib.decompress(payload).decode())
```

Flaskのsession cookieは署名で改竄検出はできるものの、**データ自体は読み取り可能**です。Payload部分をデコードすると flag が平文で含まれていることを確認できます。

Cookieの構造を理解し、デコードできるようになることが重要です。

</details>

## 演習 2: Flag for Switch（Easy）

**問題リンク**: [https://alpacahack.com/daily/challenges/flag-for-switch](https://alpacahack.com/daily/challenges/flag-for-switch)

### 問題概要

特定のUser-Agentでアクセスするとflagが得られます。

### 解法のヒント

1. ページにアクセスして、通常レスポンスを確認します
2. User-Agentヘッダーを変更してリクエストを送信してみましょう
3. curlの `-H` オプションまたは `-A` オプションを使います

<details>
<summary>詳しいヒントと解説</summary>

curlでUser-Agentを変更：

```bash
curl <サーバーのURL> -A "Switch"
```

または

```bash
curl <サーバーのURL> -H "User-Agent: Switch"
```

これはヘッダー偽装の一種である**User-Agent偽装**です。サーバーが特定のUser-Agentからのアクセスのみを許可している場合、User-Agentを変更することでアクセスできます。CTFでは「ブラウザ判定で本番環境と誤認させる」「Bot判定を回避する」など、アクセス制御の突破手段としてよく使われます。

</details>
